<h2 id="platforms-crossplay">
  Platforms, crossplay, and joining friends
</h2>

Subnautica 2 co-op supports up to four players across PC and Xbox platforms, including Steam, Epic Games Store, Windows Store / Microsoft Store, and Xbox Series X|S.

You can join through platform friends, an in-game friends list, or a Friend Code flow depending on platform and build.

```text
Host opens the world
→ enables multiplayer
→ chooses privacy
→ invites by platform friends, in-game friends, or Friend Code
```

Cross-progression is separate from guest access. Cross-progression helps your account across platforms; it does not automatically let a guest continue the host's world while the host is offline.

At Early Access launch, Subnautica 2 uses four preset playable characters. More customization is expected during Early Access. For co-op, the practical value is readability: distinct characters make it easier to recognize teammates underwater.

<h2 id="session-settings">
  Session privacy, difficulty, and PvP settings
</h2>

Before inviting friends, check the session settings screen.

Look for:

- **Session Privacy:** public, friends-only, invite-only, or code-based joining depending on build.
- **Game Mode / Difficulty:** choose Survival or Creative before the group commits to the save.
- **Friend Code / Invite Flow:** useful when platform friends do not appear correctly.
- **PvP:** Subnautica 2 is built around survival and co-op exploration, not PvP.

Use friends-only or invite-only for a serious progression save. Public access is risky because storage and base systems are shared.

<h2 id="convert-solo-save">
  Convert solo save to multiplayer
</h2>

You do not need a separate world just to play with friends.

```text
Main Menu
→ Load / select your solo world
→ Convert to Multiplayer or Host Multiplayer
→ set session privacy
→ invite friends
```

Menu wording may shift during Early Access, but the concept is stable: the world owner becomes the host and the solo save becomes the shared session world.

<h2 id="convert-back-single-player">
  Convert multiplayer back to single player
</h2>

If your friends leave and you want to continue alone, convert the session back to single player.

```text
Load the multiplayer world
→ open world / session options
→ Convert to Single Player
→ confirm and continue alone
```

This answers the common worry: converting to co-op should not trap you into always playing multiplayer.

<h2 id="share-save-host">
  Share a save copy so another friend can host
</h2>

If your connection is unstable or you are rarely online, you can share a copy of the save with a trusted teammate and let them host that copy.

Use the safe process:

```text
Host makes a backup
→ host sends the save copy to trusted friend
→ friend imports / places it in the correct save location
→ group agrees which copy is now the main world
```

Exact file paths can vary by platform and storefront, so confirm the save location before moving files. On Windows / Steam, check this path first:

```
%USERPROFILE%\Saved Games\Subnautica2
If your Steam build does not use that folder, right-click the game in Steam, open **Manage → Browse local files**, then search the game folder and Saved Games for the newest save directory. Epic, Windows Store, and Xbox builds may use different account-linked folders, so do not overwrite anything until you have made a backup.
```

Do not keep two “main” copies active, or the group will split progress.

<h2 id="host-save">
  Host responsibilities: who should host?
</h2>

The best host is not always the best player. Pick the person with the most stable connection, most consistent schedule, and best chance of keeping the save organized.

A bad host choice can stall the group even if everyone helped build the world.

<h2 id="dynamic-scaling">
  Does co-op change difficulty or resource pressure?
</h2>

Co-op changes practical difficulty even when the base mechanics are familiar. More players consume materials faster, crowd routes, use shared storage, and create more vehicle demand. Some Early Access reports also describe dynamic pressure around resource depletion and predator behavior as lobby size changes, so bigger groups should play with more structure.

<h2 id="storage-base-risk">
  Shared storage and base safety risks
</h2>

Shared storage is convenient, but invited players can use team materials and change the base.

Use project lockers:

```text
TADPOLE
BASE
TOOLS
RARE
FOOD
```

Assign one builder to approve major layout changes and one vehicle lead to decide when rare materials leave storage.

<h2 id="death-items">
  Death, dropped items, Biobeds, and revive limits
</h2>

There is no teammate revive loop to rely on. If a player dies, they respawn instead of waiting for a revive.

In co-op, dropped items can remain where the player died, and a teammate may be able to pick them up. This can save the dead player from making a second risky trip. Depending on progression and location, the dead player respawns at the nearest Biobed rather than exactly where the team is.

Use this recovery plan:

1. Call out the last known location.
2. Decide whether a teammate can safely pick up the dropped items.
3. Do not send rare materials into the same danger route twice.
4. Regroup at the nearest Biobed, base, vehicle, or named marker before continuing.

<h2 id="no-tether">
  Can players split up, or is there a tether?
</h2>

Players can split jobs instead of staying glued together. In practical Early Access play, players can separate by hundreds of meters without a strict short tether forcing everyone back together.

The farther players split, the more important the role system above becomes.

<h2 id="vehicles">
  Vehicles are team assets
</h2>

Vehicles such as the Tadpole should be treated like shared infrastructure, not personal toys.

Use vehicle rules:

- say where you are taking it,
- keep it near agreed routes or bases,
- do not leave rare materials inside without telling the team,
- build more than one vehicle if your group splits often.

If your team is working toward the first vehicle, use the <Link href="/subnautica-2/tadpole-submarine">Tadpole Submarine guide</Link>.

<h2 id="faq">
  FAQ
</h2>

### Do I need mods for Subnautica 2 co-op?

No. Subnautica 2 has native co-op support, so you do not need a multiplayer mod.

### Can I switch a multiplayer save back to solo?

Yes. Use world or session options to convert the multiplayer world back to single player, then continue alone.

### Can a friend host my save instead?

Yes, if you deliberately share a copy of the save with a trusted teammate. Confirm the correct save location for your platform or storefront first.

### Can a teammate recover my dropped items?

Yes, if they can safely reach the death location. A teammate may be able to pick up dropped items, but the team should still regroup after recovery.

### Does a new guest get my adaptations?

No. Treat DNA Adaptations and personal body upgrades as personal progression. A new guest should check their own survival setup first.
