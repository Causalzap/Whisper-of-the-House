<GuideQuickAnswer title="Quick Answer">
**A Belt Filter in Shapez 2 reacts differently to shape, 1, 0, null, and conflict signals.**

The key behavior is:

- **shape signal**: matching shapes pass; non-matching shapes go to the side output
- **1 / On**: every shape passes
- **0 / Off**: every shape goes to the side output
- **null**: the filter blocks and shapes stop
- **conflict**: the filter blocks and shapes stop

The safest way to debug wires is to place a **Display** on the wire feeding the filter. It shows what signal the filter is actually receiving, so you do not have to guess whether the problem is the signal, the filter direction, the shape, or the output path.
</GuideQuickAnswer>

<h2 id="why-belt-filter-not-working">
  Why your Belt Filter is not working in Shapez 2
</h2>

Most Belt Filter problems are caused by a signal mismatch. The filter may be fine, but the wire may be carrying 1, 0, null, conflict, or the wrong shape signal.

<GuideTable
  caption="Shapez 2 Belt Filter not working troubleshooting."
  columns={notWorkingColumns}
  rows={notWorkingRows}
/>

A useful debugging order is:

1. Put a Display on the wire.
2. Check whether the signal is shape, 1, 0, null, or conflict.
3. Check whether the input shape matches the signal.
4. Check filter direction.
5. Check the straight output.
6. Check the side output.
7. Check whether a Belt Reader is going null between shapes.

<h2 id="signal-types">
  Shape signal vs on/off signal vs null in Shapez 2
</h2>

A common beginner mistake is treating every signal as the same kind of signal.

<GuideTable
  caption="Shapez 2 signal types for Belt Filters and wire logic."
  columns={signalTypeColumns}
  rows={signalTypeRows}
/>

For a beginner Belt Filter test, a fixed 1 or 0 signal can tell you whether the filter is receiving a signal at all. But a real shape-sorting setup usually needs a shape signal or comparison signal so the filter knows what to accept.

If the filter receives the wrong kind of signal, it may behave like the wire is broken even though the connection is fine.

<h2 id="belt-reader-null-output">
  Why Belt Readers output null between shapes
</h2>

A Belt Reader only outputs a shape signal while a shape is physically on the reader. When no shape is currently on it, the signal becomes **null**.

That matters because a Belt Filter receiving null blocks. On a sparse belt, the reader can output a shape signal for a moment, then null between items, then another shape signal. If you connect that directly to a filter, the filter may appear to work only sometimes.

Use this test:

1. place a Display after the reader
2. watch the signal while shapes pass
3. check whether it flickers between shape and null
4. only then connect it to a Belt Filter

If you see null gaps, the reader is not broken. It is doing exactly what it does when no shape is on it.

<h2 id="display-debugging">
  How to use Displays to debug wires
</h2>

Displays are the fastest way to debug wire problems because they show the current wire signal.

Use a Display whenever:

- a Belt Filter blocks everything
- a Belt Filter lets everything through
- a Belt Reader seems inconsistent
- two signals may be conflicting
- you are not sure whether a wire contains a shape signal or on/off signal
- a MAM module works in one case but fails in another

<h2 id="when-to-use-logic-gates">
  When to use logic gates in Shapez 2 wires
</h2>

Use logic gates only after a simple filter works.

<GuideTable
  caption="Common Shapez 2 logic gate traps."
  columns={logicTrapColumns}
  rows={logicTrapRows}
/>

The most common trap is using **OR** to combine two shape signals. OR does not create a "red circle or white circle" shape signal. It outputs **1**, and a Belt Filter receiving 1 allows every shape through.

Use logic gates for multi-condition systems after you understand what signal each wire actually contains.

<h2 id="faq">
  FAQ
</h2>

### How do Wires work in Shapez 2?

Wires carry signals between automation components. Those signals can control filters, gates, comparisons, and logic decisions.

### What does a Belt Filter do?

A Belt Filter reacts to its input signal. Shape signals pass matching shapes and redirect non-matching shapes; 1 passes everything; 0 redirects everything; null and conflict block the filter.

### Why is my Belt Filter blocking everything?

The filter is probably receiving null or conflict. Place a Display on the wire feeding the filter to confirm the signal.

### Why is my Belt Filter letting everything through?

The filter is probably receiving a 1 / On signal. A 1 signal passes everything; it does not filter for a specific shape.

### Why does my Belt Reader work only sometimes?

A Belt Reader outputs the shape signal only while a shape is physically on the reader. Between shapes, the signal becomes null, which can make a connected Belt Filter block.

### What does a Display do?

A Display shows the current signal on a wire. Use it to debug whether the wire contains a shape signal, 1, 0, null, or conflict.

### Why should I not use OR to combine shape signals?

OR outputs 1 / On, not a combined shape signal. If you feed that into a Belt Filter, every shape passes.

### When should I use logic gates?

Use logic gates after a simple filter works. They are for multi-condition systems, not for your first wire setup.

### Do I need Wires before building a MAM?

Yes. A MAM depends on target reading, filtering, routing, reject handling, and signal debugging.
