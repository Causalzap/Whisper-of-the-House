<GuideQuickAnswer title="Quick Answer">
**A MAM in Shapez 2 is a Make Anything Machine: a factory that reads a target shape signal and automatically builds that shape using wires, Global Receivers, simulated buildings, Belt Filters, reusable modules, and cleanup routing.**

For Shapez 2 **1.0**, the safest first MAM is not an ultimate four-layer blueprint. Start with a small 1.0-tested loop:

1. receive or read the target signal
2. show it on a Display
3. process the signal with simulated buildings
4. normalize quadrant logic to NE
5. route one correct part with a Belt Filter
6. build one physical operation module
7. verify output, trash bad shapes, then scale

Old Early Access MAM blueprints can still teach good ideas, but in 1.0 they should be retested or rebuilt before you trust them.
</GuideQuickAnswer>

<h2 id="what-is-a-mam">
  What is a MAM in Shapez 2?
</h2>

A **MAM**, or **Make Anything Machine**, is a factory that can make different requested shapes without you rebuilding the factory each time.

In 1.0, a useful MAM has four layers of logic:

1. **Target logic** — receive or read what the factory should make.
2. **Signal logic** — use simulated buildings to inspect, split, rotate, and compare the shape signal.
3. **Routing logic** — send the right physical parts to the right modules.
4. **Production logic** — cut, rotate, paint, stack, pin, crystallize, clean up, and output the final shape.

The name is slightly misleading for beginners. A first MAM does not need to make literally everything. It only needs to prove that your factory can read a target and produce a matching output automatically.

<h2 id="shapez-2-1-0-mam-notes">
  Shapez 2 1.0 MAM notes
</h2>

This guide is written for **Shapez 2 1.0**, not for older Early Access-only blueprint assumptions.

The important 1.0 takeaway is:

- use old MAM videos and blueprints as ideas
- retest every copied module
- rebuild signal-heavy blueprints when needed
- confirm Global Receivers, channels, pins, crystals, simulated rotations, and layer routing in your current save
- do not assume a pre-1.0 blueprint still handles 1.0 targets correctly

A release-week MAM example explicitly planned to throw away older blueprints and rebuild them for 1.0. That does not mean every old idea is wrong, but it does mean you should validate the module before trusting it.

<h2 id="global-receiver-target-signal">
  How Global Sender and Global Receiver work in a Shapez 2 MAM
</h2>

Global Sender and Global Receiver are the pieces that let a MAM react to a target without running a local wire across the whole factory.

  caption="Global signal tools for Shapez 2 MAM target reading."
  columns={globalSignalColumns}
  rows={globalSignalRows}

For a first real MAM, the important idea is:

1. receive the objective / target shape signal
2. show it on a Display
3. split or rotate the signal with simulated buildings
4. use the resulting signal to control physical modules

<h2 id="simulated-buildings">
  Simulated Buildings in Shapez 2 MAMs
</h2>

Simulated buildings are one of the biggest differences between a simple factory and a real MAM.

They do not cut, rotate, or stack physical shapes. They operate on the **shape signal**. This lets the MAM inspect the target before deciding what the physical factory should do.

<GuideTable
  caption="Simulated buildings and signal tools for Shapez 2 MAM logic."
  columns={simulatedBuildingColumns}
  rows={simulatedBuildingRows}
/>

The signal layer runs before the physical factory moves any real shapes. The Simulated Unstacker separates layer information from the target signal, the Simulated Rotator normalizes each piece to NE, and the Shape Analyzer reads color or part type before a physical module is triggered.

That distinction matters: simulated buildings decide **what should be built**, while cutters, rotators, painters, stackers, pins, and crystal modules actually build it.

<h2 id="one-layer-or-one-quadrant-first">
  Start with one layer or one quadrant first
</h2>

The simplest first MAM should not solve every shape. It should solve the smallest target scope that still teaches the real MAM loop.

<GuideTable
  caption="How to choose your first Shapez 2 MAM target scope."
  columns={firstScopeDecisionColumns}
  rows={firstScopeDecisionRows}
/>

A practical starting point is: **one layer, one normalized NE quadrant, no pins, no crystals, and one visible output check**.

This makes failures easier to locate. If a complete four-layer MAM outputs the wrong shape, the bug could be in the target signal, simulated split, rotation, layer route, physical module, or final stack. If a one-quadrant MAM outputs the wrong shape, the bug has fewer places to hide.

Copying a MAM is not the end of the job.

The most important rule is to separate the control channels. If one MAM uses one target system and the second MAM is supposed to do task shapes, operator shapes, or a separate goal, the copied MAM needs its own channel identifiers.

<h2 id="faq">
  FAQ
</h2>

### What does MAM mean in Shapez 2?

MAM means Make Anything Machine. It is a player term for a factory that can automatically build many requested shapes by reading a target and routing parts through reusable modules.

### Is MAM an official building?

No. A MAM is not one building. It is a factory design made from Global Receivers, readers, simulated buildings, wires, Displays, Belt Filters, physical operation modules, trash routes, and transport.

### What does a Global Receiver do in a MAM?

A Global Receiver can receive the current objective or target shape signal and feed it into the MAM, so the factory can switch targets without manually rebuilding the signal every time.

### What are simulated buildings in Shapez 2?

Simulated buildings process the shape signal on the wire layer. For example, a Simulated Unstacker can split a target shape signal into layers, and a Simulated Rotator can normalize a quadrant before comparison.

### Why should I normalize to NE?

Treating northeast as the default orientation makes quadrant comparison and simulated rotation logic easier. Simulated Rotators measure the signal relative to a reference direction, so the same quadrant in a different orientation can produce a different signal and break comparison logic that worked on your test shape. Normalize first, then analyze or route.

### Should my first MAM support every shape?

No. Your first MAM should support a small target family, one layer, or one quadrant. Add pins, crystals, multiple layers, and multiple machines later.

### Do I need Wires before building a MAM?

Yes. A MAM depends on target signals, filter routing, simulated buildings, Displays, rejects, and logic. Learn the <Link href="/shapez-2/wires-belt-filter-guide">Wires and Belt Filter Guide</Link> first.

### Do old MAM blueprints work in Shapez 2 1.0?

Treat old blueprints as references, not guaranteed working builds. Retest or rebuild each module in 1.0, especially if it uses wires, pins, crystals, Global Receivers, simulated buildings, or copied control channels.

### Why does my MAM fail on pins?

Pins do not behave like normal colored quadrants. If your logic expects a color signal, a pin can break that path. Add a separate pin check or pin route.

### Why does my MAM output the wrong orientation?

A module may have been rotated, mirrored, or copied with the wrong orientation, or the simulated signal was not normalized to NE before comparison. Find the first wrong intermediate output and fix that module.

### Can I copy a MAM to make it faster?

Yes, but copying is not enough. You must change control channels, receivers, outputs, and supply routing so the copied MAM does not reuse the wrong signals.

### What is the best MAM use in Manufacture Mode?

Operator shape automation is usually the most practical early use because operator shapes support research points and platform capacity. A task-shape MAM is useful later when your first MAM is stable.
