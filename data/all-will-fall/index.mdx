<h2 id="best-guide-order">
  Best guide order
</h2>

If you are not sure which page to open first, use this order:

### 1. Open the scenario you are actually stuck on

That sounds obvious, but it is still the right first rule.  
If you are dying in **Oil Rig**, a better Tornado Race plan will not save that run.

### 2. Open collapse troubleshooting when structure problems are blocking progress

The collapse page is the best cross-scenario support page in the cluster because structural mistakes travel across multiple maps.

### 3. Return to the scenario page after fixing the bottleneck

This cluster works best when used like this:

- scenario page for route and priorities
- collapse page for structural diagnosis
- back to the scenario page for execution

<h2 id="which-guide-should-you-read-first">
  Which guide should you read first?
</h2>

### Read Oil Rig first if:

- you want the clearest practical example of a scenario page
- you care about early build order and visible water optimization
- you prefer a more direct survival-and-layout guide

### Read Tanker Truck first if:

- you care more about build order math and movement economy
- you are struggling with zone transitions
- you want a route built around continuity rather than one fixed base

### Read Tornado Race first if:

- you are dealing with a hard deadline
- you want phase-based planning
- you need a platform-first scenario guide rather than a comfort-first colony guide

### Read collapse troubleshooting first if:

- your real bottleneck is structural failure
- the scenario page makes sense, but your buildings still keep falling apart
