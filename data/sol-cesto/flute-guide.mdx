<h2 id="what-the-flute-actually-does">
  What the Flute actually does in Sol Cesto
</h2>

The Flute is a **hidden utility system**.

It is not just a collectible, and it is not just lore flavor.

Once assembled, it lets you play melodies that create specific run effects, including:

- **shop discount**
- **sending money home**
- **healing**
- **switching Physical and Magic attack**

That makes the Flute a real run-impacting tool, not a side curiosity.

<GuideCallout title="Important distinction">
The Flute is **not** part of the Egg / Lizard unlock path.

If you are trying to unlock a hidden character, this is the wrong system.

If you are trying to understand a hidden run tool with melody effects, this is the right one.
</GuideCallout>

---

<h2 id="how-many-flute-fragments-you-need">
  How many Flute Fragments you need
</h2>

You need **4 Flute Fragments** to complete the Flute.

That is the first thing most players need clarified, because a single Fragment does not explain the system.

The full loop is:

1. find **4 Flute Fragments**
2. assemble the Flute
3. play a melody for an effect
4. the Flute breaks
5. repair it with the fountain bird
6. use another melody later in the run

<h2 id="how-to-use-the-flute">
  How to use the Flute
</h2>

Once the Flute is assembled, the basic use loop is simple:

1. enter the melody input
2. play a valid melody
3. get the effect
4. the Flute breaks
5. later repair it with the fountain bird
6. use another melody if you get the repair

That means the Flute is best treated like a **high-value limited-use tool**, not a background collectible.

---

<h2 id="how-flute-notation-works">
  How Flute notation works
</h2>

The Flute uses color inputs with **note length**.

The notation rule is:

- **UPPERCASE** = long note / long press
- **lowercase** = short note / short press

So a sequence like:

**RED RED red blue yellow**

does not just mean “five colors.”

It means a specific mix of long and short notes.

<GuideCallout title="Most common input mistake">
If a melody is not working, the problem is often **note length**, not the color order.
</GuideCallout>

<h2 id="what-each-melody-does">
  What each melody does
</h2>

### 1. Shop Discount (Beyond Worlds)

**Input:** `blue red YELLOW BLUE YELLOW`

**Effect:** gives you a **shop discount**.

**Where the clue comes from:** players trace this melody to the **Steam store page gif clue**, which is one reason it feels so hidden compared with normal in-game hints.

**Why it matters:** this is one of the easiest melodies to justify because better shop value helps almost any run.

---

### 2. Send Money Home (Melody in the Box)

**Input:** `RED RED red blue yellow`

**Effect:** sends your money **home**.

**What “home” means here:** it sends your coins back into your **between-run progression economy**, instead of leaving that value tied to the current run.

**How to unlock it:** after assembling the Flute, a **mysterious music box** can appear in the shop. Buying it for **20 coins** reveals the melody.

**Why it matters:** this is one of the strongest utility melodies because it turns a good run into safer long-term progression.

---

### 3. Healing

**Input:** `YELLOW red blue RED BLUE`

**Effect:** healing.

**How to unlock it:** players report unlocking this melody by **throwing an ice cube into the pond**.

### 4. Switch Physical and Magic Attack (Blacksmith Code)

**Input:** `blue RED blue RED yellow`

**Effect:** switches **Physical and Magic attack**.

**How to unlock it:** players report revealing this melody in the **blacksmith upgrade room** by using a **Candle** to expose the hidden code.

**Important tradeoff:** this is not just a free room secret.

To reveal it, you need to bring a **Candle** into the blacksmith upgrade route, which means committing a run resource and setup choice to hidden-system progress instead of using that slot or item path elsewhere.

There is not yet a fully clean public confirmation on whether the Candle is always consumed by this interaction, so the safest confirmed tradeoff is the **setup cost**, not a guaranteed item loss.

**Why it matters:** this is the most obviously “hidden-tech” melody because it depends on understanding a very specific room interaction.

<h2 id="does-the-flute-break-after-use">
  Does the Flute break after use?
</h2>

Yes.

After you successfully play a melody, the Flute **breaks**.

That is a core part of the system.

The Flute is not designed as a permanently reusable free-action item.

It is designed more like a high-value limited-use tool.

---

<h2 id="how-bird-repair-works">
  How bird repair works
</h2>

The system does not end when the Flute breaks.

Official 1.0 update text confirms that the **bird in the fountain** repairs the broken Flute so you can play a new melody afterward.

### Where the bird is

The bird is tied to the **fountain**.

### Is the bird guaranteed to appear?

No reliable public evidence shows that the bird appears **every single time** you reach a fountain.

The safest current reading is that the **bird is one possible fountain event**, not a permanent always-available repair station.

That means bird repair should be treated as:

- a **confirmed repair mechanic**
- but also a **conditional encounter**, not something you can force on demand every run

### How repair works

The practical loop is:

1. assemble the Flute
2. play a melody
3. the Flute breaks
4. later find a fountain event with the bird
5. interact with the bird
6. the Flute is repaired
7. use another melody afterward

### Is there a separate repair fee?

There is **no clearly confirmed separate coin fee** attached to the repair.

The best-supported reading is that the repair is simply the **bird event itself**, not a shop-style paid service.

<GuideCallout title="Bird repair, simplified">
The bird is best understood as a **fountain repair event**, not a merchant transaction.

What is confirmed is the repair itself. What is **not** fully confirmed is that the bird appears at every fountain.
</GuideCallout>
