<GuideQuickAnswer title="Quick Answer">
**BOMBANANA** is a three-player bomb-defusal co-op game where Blind, Deaf, and Mute monkeys must solve modules before time or mistake limits run out.

Your real advantage is not memorizing every answer first. It is using better callouts than other teams: **module name → position → required details → manual answer → repeat → confirm → input**. The biggest beginner mistake is acting before the team knows which module, LED, number, direction, or switch rule is actually being solved.

<h2 id="communication-flow">
  Communication flow before you defuse
</h2>

Use one callout structure for every module. The exact details change by module, but the opening rule never changes: **say the module name and position first**.

<ol className="my-6 list-decimal space-y-2 pl-6">
  <li>Say the module name.</li>
  <li>Say the module position.</li>
  <li>Give only the details that module needs.</li>
  <li>Let the manual holder solve and signal the answer.</li>
  <li>Repeat the answer as color, number, direction, switch state, or position.</li>
  <li>Wait for confirmation.</li>
  <li>Cut, press, type, switch, or enter.</li>
</ol>

A good team sounds boring when it is working. That is the goal. BOMBANANA becomes easier when every player knows what kind of sentence comes next.

If the module changes after a wrong input, restart the callout. Do not reuse the old LED and number.

  Numpad, upper left. Expression is 52 minus 25. Result is 27, odd. Green LED after entry. Waiting for target number.
</blockquote>

The handler should then find the target number wherever it appears. Do not assume the keypad is arranged like a normal keypad.

<h2 id="numpad-odd-even-greater-less-priority">

A real point of confusion is that a number can satisfy more than one description. For example, a result can be **odd** and also **greater than 5**. That does not mean the bomb handler should choose whichever condition sounds right.

The safe rule is: **the manual holder chooses the branch; the bomb handler only reports the state and presses the final target**.

<h2 id="faq">
  BOMBANANA demo FAQ
</h2>

### What is BOMBANANA?

**BOMBANANA** is a three-player co-op bomb-defusal game where Blind, Deaf, and Mute monkeys must solve bomb modules with limited communication.

### How many players do you need for BOMBANANA?

The demo structure is built around three roles: Blind, Deaf, and Mute. The game works best when each player accepts their communication restriction instead of trying to bypass it.

### How many levels are in the BOMBANANA demo?

The demo is reported as seven difficulty tiers. Early levels teach wires and basic communication, while later levels add D-pad, numpad/math, switch panels, and heavier multi-module pressure.

### Should we use external voice chat?

No. The tutorial specifically tells players not to use external voice chat. The point is to solve modules through the game's role-limited communication.

### What should we communicate first?

Always start with the module name and module position. Then give the details the manual holder needs, such as wire count, LED color, center number, math expression, or switch color order.

### How do wires work in BOMBANANA?

For wires, communicate the number of wires, the wire colors from left to right, and the LED color. Do not cut until the answer is repeated and confirmed.

### How does the D-pad module work?

For the D-pad, communicate the LED color and center value, then use the shared left/right perspective to press the correct direction. If the module changes after a mistake, call the new state again.

### How does the numpad module work?

For numpad modules, solve the visible math expression, identify odd or even if needed, read the LED state, handle comparison clues if they apply, then press the target number given by the manual logic. The keypad may be jumbled, so press the number itself, not a normal keypad position.

### If a numpad result is both odd and greater than 5, which rule wins?

Do not let the bomb handler guess. The manual holder should use the LED color and the manual's branch order to choose the final target number. The handler should report the full state, such as result 67, odd, greater than 5, yellow LED, then wait for the final key.

### How does the switch panel work?

For switch panels, communicate the top light color order, the full digit string, and each switch state. Press enter only after all switch positions are confirmed.

### What does the Blind role do?

The Blind player handles the bomb, locates modules by touch, and may read or interpret tactile information such as dots or braille-style values.

### What does the Deaf role do?

The Deaf player can participate in communication but cannot rely on hearing the team, so visual confirmation and clear gestures matter.

### How many levels are in the BOMBANANA demo?

The demo has **seven difficulty tiers**. Early levels focus on wires and basic role communication, while the middle levels start adding D-pad, numpad, and math logic. Around Level 4–5, the numpad becomes more important because results can involve **odd/even** and **greater-than/less-than** style decisions instead of just typing the raw answer.

### What does the Mute role do?

The Mute player holds the defusal instructions and must use gestures instead of voice, so the team should agree on basic signs before starting.

### Why do we keep failing the first modules?

Most early failures come from incomplete callouts, not hard puzzles. Say the module name, wire count, LED color, number, or stage before asking for an answer.

### What should we do after a wrong input?

Re-read the module state. The LED, number, stage, or required answer may have changed after the mistake.

### Is BOMBANANA good for Steam Next Fest?

Yes, if your group likes short, chaotic, communication-heavy demos. It is especially strong as a party-game demo because every failure teaches a better team rule.
