<h2 id="resume-rules-thanks-letter-guide">
  Thank You For Your Application resume rules and Thanks Letter guide
</h2>

This **Thank You For Your Application resume rules guide** explains how to approve, reject, and write the correct Thanks Letter reason during workdays. The goal is simple: make the right hiring decision and attach the right rejection reason before the game turns small mistakes into wage penalties or route damage.

This page focuses on document screening. For full endings, use the <Link href="/thank-you-for-your-application/endings-guide">Endings Guide</Link>. For all Steam achievements, use the <Link href="/thank-you-for-your-application/achievements-guide">Achievements Guide</Link>.

<h2 id="safe-workday-routine">
  Safe routine for every workday
</h2>

Standard Mode is timed. Time runs during interviews, and no more candidates can be called after the end-of-day cutoff. Subsidy Mode removes the interview time barrier, but the document logic is the same.

Use the same check order every shift. In Standard Mode, this routine only feels slow at first; after a few days, the fixed order helps you scan faster because you are no longer deciding what to check next.

<h2 id="local-vs-non-local-candidates">
  Local vs non-local candidates
</h2>

Local vs non-local status changes which extra documents you should expect. Do not guess from the candidate portrait or dialogue. Read the school / city on the resume, then compare it with the current Job Requirements and School Reference Table.

Early non-local checks use schools such as **University of Manato**, **Manato Ocean University**, and **Manato Community College** as obvious warning signs. If a candidate is treated as non-local that day, look for the required extra document before approving.

<h2 id="thanks-letter-ui">
  Thanks Letter UI: select reason, generate, attach, reject
</h2>

Once Thanks Letters are required, rejecting is a four-step action: select the failed field, press the Thanks Letter button at the lower-right area of the interface, attach the generated ticket to the resume, then send the resume to the red reject folder.

The button only matters after you select the exact reason. If the report date is wrong, select the date. If the certificate is missing, select the certificate requirement. If the social number does not match, select the mismatched identity field.

<h2 id="breakdown-index-limits">
  Breakdown index limits
</h2>

Do not mix up two different breakdown checks. **Candidate breakdown index** is a document-screening value on Psychological Evaluation Reports. **C89's own breakdown index** is your survival / route-safety value, checked through apartment status or A02.

Candidate breakdown decides whether the applicant passes today's rule. C89's own breakdown does not change the correct hiring decision, but it can make overtime, sleep, and late route choices dangerous.

<h2 id="mental-health-and-document-checking">
  Does your own mental health affect document checking?
</h2>

C89's own breakdown index does not change which candidate rule is correct. If a candidate fails the school rule, certificate rule, internship rule, or report-value rule, the correct document decision stays the same.

What your own mental state changes is route safety. High breakdown makes overtime and end-of-day choices more dangerous. Before you keep working late, shop, sleep, or push a route scene, check personal status through A02 or the apartment status screen.

<h2 id="when-to-approve">
  When should you approve a candidate?
</h2>

Approve only when every active rule passes:

<ul className="my-6 list-disc space-y-2 pl-6">
  <li>the required resume is present,</li>
  <li>school / local / non-local status passes today's rule,</li>
  <li>all required extra documents are present,</li>
  <li>identity fields match across documents,</li>
  <li>dates and stamps are valid,</li>
  <li>internship or work-experience requirements are met,</li>
  <li>Psychological Evaluation Report rules pass,</li>
  <li>and breakdown-index value is within today's allowed limit.</li>
</ul>

If any one of these fails, do not approve. Select the exact failed reason and generate the Thanks Letter.

<h2 id="faq">
  Thank You For Your Application resume rules FAQ
</h2>

### What is the first thing I should check?

Open the Job Requirements tab. It shows which school, document, internship, report, and breakdown-index rules are active that day. Read it before calling any candidate.

### Where is the Thanks Letter button?

After you select the failed requirement or invalid field, use the Thanks Letter button at the lower-right area of the interface. Generate the ticket, attach it to the resume, then reject.

### Why did I lose money even though the candidate should fail?

The pass / fail decision may be correct, but the Thanks Letter reason may be wrong or missing. Once Thanks Letters are required, the rejection reason matters.

### How do I tell whether a candidate is local or non-local?

Read the school / city shown on the resume and compare it with the School Reference Table and today's Job Requirements. Early examples treat Aeropolis / Aropolis schools as local and Manato schools as non-local.

### Do non-local candidates always need a graduation certificate?

Do not assume it forever. Check today's rule. When the rule says non-local graduates need a graduation certificate, reject non-local candidates who do not provide it.

### What is the breakdown index limit?

Candidate breakdown-index limits can change by day. Early report checks can use 40%, and later workdays can show 35. C89's own breakdown danger also uses 40 on Day 4 and 35 after Day 4.

### Is missing Psychological Evaluation Report the same as invalid date?

No. If the report is missing, select the missing report requirement. If the report exists but the date is invalid, select the report date field.

### Does C89's own breakdown index change document rules?

No. It does not change which candidate rule is correct. It affects route safety, overtime risk, and whether ending the day is dangerous.

### Does Standard Mode have a time limit?

Yes. Standard Mode is timed: time runs during interviews and the workday cuts off at the end-of-day limit. Subsidy Mode removes the interview time barrier.

### Should I use this page or the Beginner Guide?

Use this page when you are making document decisions or choosing Thanks Letter reasons. Use the Beginner Guide for broader survival topics like pay, bills, Subsidy Mode, A02, and first-run planning.
